# App Store 上架審查 - 自動循環模式

請你扮演 App Store 審核官 + 資深 iOS/Android 開發者。

針對這個專案，請執行以下循環：

## 1. 審查
依照 Apple App Store Review Guidelines 逐項檢查，包含但不限於：
- **效能與穩定性**：崩潰、記憶體洩漏、載入速度、大量資料處理
- **UI/UX 設計**：HIG 規範、Material Design、無障礙(Accessibility)、多語系、深色模式
- **安全性**：API 金鑰暴露、資料加密、HTTPS、隱私政策、Token 安全儲存
- **功能完整性**：錯誤處理、空狀態畫面、離線模式、網路斷線提示
- **法規合規**：金融相關免責聲明、投資風險警語、個資保護聲明
- **App Store 必要項目**：App Icon 全尺寸、啟動畫面、截圖、App 描述、隱私權政策頁面、年齡分級
- **程式碼品質**：測試覆蓋率、無 hardcode 敏感資訊、錯誤日誌、程式碼結構

## 2. 評分
給出目前的上架準備度（0-100 分），並列出所有不合格項目，依嚴重程度排序：
- **🔴 阻擋上架**：必須修復才能送審
- **🟡 高風險**：很可能被退件
- **🟢 建議改善**：提升使用者體驗

## 3. 修復
從最嚴重的問題（🔴）開始修復，每修一個就 commit + push

## 4. 重新審查
修完後重新跑一次步驟 1，檢查是否產生新問題

## 5. 重複
直到評分達到 95 分以上才停止，每輪結束報告：
- 目前分數
- 本輪修復了什麼
- 剩餘問題清單

## 注意事項
- 過程中不要停下來詢問，直接持續修復
- 遇到以下情況才暫停：需要 API Key、需要第三方服務帳號、重大架構決策、需要付費服務
- 每次修復都要確保不會破壞現有功能
